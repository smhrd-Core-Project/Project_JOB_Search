  ## 🚀 프로젝트 소개

### 📌 프로젝트 개요
- **팀명**: [틴넥트 (Teennect)]
- **설명**  
  > [청소년을 대상으로 한 진로 설계, 개인의 흥미와 성향에 맞춘 학과/직업 추천 서비스]
- **기간**: [2025.05.15 ~ 2025.05.29]
- **노션링크**: (https://slime-roast-ee5.notion.site/1cba64f7def180859dcac7242f424fad))
#
  
### 🎯 배경 및 목표

1. **문제 정의**  
   - 중·고등학생 중 약 40%가 장래희망이 “없다”고 응답했으며, 이들 중 73.1%가 “무엇을 할 수 있을지 몰라서” 진로 결정을 하지 못함  
   - 설문 기반 진로 탐색 과정이 복잡하고 지루해 50% 이상이 “절차가 복잡하다”거나 “정보를 이해하기 힘들다”고 느끼고 있음  
   - 학생들은 진로 선택 시 ‘나의 적성’을 가장 중요한 기준으로 꼽지만, 기존 서비스는 이를 직관적으로 파악·추천해주지 못함  

2. **목표**  
   - **진로 탐색 절차 간소화**: 설문 설계를 단순화하고 재미 요소를 가미해 누구나 부담 없이 시작하도록 함  
   - **적성 발견 지원**: AI(=KR-SBERT 등)를 활용해 학생 개개인의 특성과 흥미를 빠르게 분석  
   - **맞춤형 진로 추천**: 분석 결과를 바탕으로 나에게 딱 맞는 전공·직업을 제시하여 진로 결정 장벽을 낮춤  


#

### 💫 주요기능
- **회원 인증/권한 관리**: 네이버 Open API를 활용한 소셜 로그인, 추천받은 정보는 카카오 Kakao Open API 활용하여 원하는 사람에서 공유
- **설문조사 & AI 추천**: 파이썬(PYTHON)KR-SBERT를 활용한 전공 유사도 계산
- **커뮤니티 게시판**: 글 작성·댓글·좋아요·조회수 집계

### 🛠️ 트러블 슈팅

**1.자연어 기반 모델 도입이유** 

<p align="center">
  <img src="https://github.com/user-attachments/assets/b491816c-e61f-44f7-89c7-20d09aa7559a" alt="초기 벡터화 문제" width="100%" style="margin-right:8px; border-radius:4px;"/>
  <!-- SBERT 도입 후: 의미 기반 임베딩으로 안정화 -->
  <img src="https://github.com/user-attachments/assets/e5a334d4-6647-4ff9-9d4d-4f6bf98ff59a" alt="SBERT 적용 후" width="100%" style="border-radius:4px;"/>
</p>

 **문제 현상**  
   - 각 학과별 키워드를CountVectorizer/TF-IDF로 벡터화했더니,  
     특정 학과에 값이 과도하게 편중되어 비교가 어려웠음  
   - 짧은 문장·동의어 미처리로 인해 의미적 유사도 손실 발생  

 **원인 분석**  
   - TF-IDF는 단어 빈도에만 의존 → 문맥을 반영하지 못함  
   - 전처리(불용어 제거, 형태소 분석 등) 단계가 미흡하여 벡터 퀄리티 저하  

**해결 방안**  
   - **SBERT** 기반 문장 임베딩 도입  
   - 코사인 유사도 기반 필터링 로직 추가  
   - (옵션) **차원 축소**(PCA/t-SNE) 로 시각화 시 가독성 확보  

**결과**  
   - 의미적 일관성이 확보된 벡터 분포 획득  
   - 추천 정확도 약 **15%** 향상 (A/B 테스트 기준)  


**2.각 테이블 트리거 문제** 
<p align="center">
  <img src="https://github.com/user-attachments/assets/433fec5f-b2cf-4a8c-adc6-5fd6ffa249a1" alt="초기 벡터화 문제" width="100%" style="margin-right:8px; border-radius:4px;"/>
  <!-- SBERT 도입 후: 의미 기반 임베딩으로 안정화 -->
  <img src="https://github.com/user-attachments/assets/096eecdc-d282-432d-b9bc-9856db2fc25d" alt="SBERT 적용 후" width="100%" style="border-radius:4px;"/>
</p>

**문제 현상**
- 각 학과별 키워드를 CountVectorizer/TF-IDF로 벡터화했더니, 특정 학과에 값이 과도하게 편중되어 다른 학과들과 비교하기 어려웠습니다. 더불어 설문 문항이 짧고 동의어 처리가 미흡해 의미적 유사도가 제대로 반영되지 못했습니다.

**원인 분석**
- TF-IDF 방식은 단어의 출현 빈도에만 의존하기 때문에 문맥이나 문장 구조를 반영하지 못합니다. 또한 불용어 제거, 형태소 분석 등 전처리 단계가 충분히 이루어지지 않아 노이즈가 포함된 단어들이 벡터에 그대로 반영되면서 품질이 저하되었습니다.

**해결 방안**
- 먼저 SBERT 기반 문장 임베딩을 도입하여 문장 단위 의미를 효과적으로 캡처하도록 했습니다. 이후 코사인 유사도를 활용한 필터링 로직을 추가해 실제 의미적으로 유사한 학과끼리 묶이도록 개선했습니다. 필요에 따라 PCA나 t-SNE 같은 차원 축소 기법을 적용하면, 시각화 시 군집 간 구분이 한층 명확해져 가독성도 확보할 수 있습니다.

**결과**
- 이 과정을 통해 의미적 일관성이 확보된 벡터 분포를 얻었으며, A/B 테스트 기준으로 추천 정확도가 약 15% 향상되었습니다
